# AI 功能调试指南

**问题**: AI 解析后内容为空  
**更新日期**: 2026-04-11

---

## 🔍 问题诊断

### 1. 检查浏览器控制台

打开浏览器开发者工具（F12），查看：

**Console 标签**:
- 是否有错误信息？
- API 调用是否成功？
- 返回的数据结构是什么？

**Network 标签**:
- 找到 `/api/parse` 请求
- 查看 Request Payload（发送的数据）
- 查看 Response（返回的数据）
- 检查 HTTP 状态码（应该是 200）

### 2. 检查后端日志

在后端终端查看：
- 是否收到解析请求？
- AI API 调用是否成功？
- 返回的 JSON 是否正确？

### 3. 常见问题

#### 问题 A：API Key 无效
**症状**: 提示 "API Key 无效" 或 401 错误

**解决方案**:
1. 检查 API Key 是否正确复制
2. 确认 API Key 是否有效（未过期）
3. 检查 API 提供商是否选择正确
4. 重新生成 API Key

#### 问题 B：文件内容为空
**症状**: 解析返回空数组

**解决方案**:
1. 检查文件是否有内容
2. 确认文件格式是否支持
3. 查看浏览器控制台的 fileContent 是否正确读取
4. 尝试使用更简单的测试文件

#### 问题 C：AI 无法识别
**症状**: 返回数据但 tasks 为空

**解决方案**:
1. 文件格式不标准 - 使用标准表头
2. 内容太复杂 - 简化文件内容
3. 语言问题 - 使用中文或英文标准术语
4. 尝试不同的模型

1. 不使用 AI 解析
2. 手动创建项目
3. 逐个添加任务

### 方案 2：使用 CSV 导入

1. 将 Excel 另存为 CSV
2. 确保格式标准
3. 重新上传

### 方案 3：简化文件

1. 只保留必需列：任务名称、开始日期、结束日期
2. 删除合并单元格
3. 删除空行和注释
4. 使用标准日期格式

## 📊 数据格式要求

### Excel 文件要求

1. **表头行**: 第一行必须是列名
2. **日期格式**: 
   - 推荐：YYYY-MM-DD（2026-04-15）
   - 支持：YYYY/MM/DD（2026/04/15）
   - 支持：YYYY年MM月DD日（2026年4月15日）
3. **无合并单元格**: 避免使用合并单元格
4. **无空行**: 删除数据中的空行
5. **编码**: UTF-8 编码

### 列名建议

**任务名称**:
- 任务、任务名称、工作项、活动、Task、Activity

**开始日期**:
- 开始日期、开始时间、起始日期、Start Date、Begin

**结束日期**:
- 结束日期、结束时间、完成日期、End Date、Finish

**负责人**:
- 负责人、责任人、执行人、Assignee、Owner

**阶段**:
- 阶段、模块、分组、Phase、Stage

## 🐛 已知问题

### 问题 1：日期识别不准确

**原因**: 日期格式不标准

**解决**: 
- 使用 YYYY-MM-DD 格式
- 避免使用相对日期（如"下周一"）
- 避免使用模糊日期（如"4月"）

### 问题 2：中文识别问题

**原因**: 模型对中文支持不够好

**解决**:
- 使用 Claude Sonnet 4（最新模型）
- 使用标准中文术语
- 避免使用缩写和俚语

### 问题 3：复杂表格识别失败

**原因**: 表格结构太复杂

**解决**:
- 简化表格结构
- 删除合并单元格
- 一个工作表只包含一个任务列表
