# GanttFlow 开发计划

## 项目概览
- 项目名称：GanttFlow - 在线甘特图协作工具
- 开发周期：18 周（约 4.5 个月）
- 团队建议：前端 2人 + 后端 2人 + UI/UX 1人 + QA 1人

---

## 阶段一：M1 - MVP 核心功能（第 1-6 周）

### 第 1-2 周：项目初始化与基础架构

#### 前端任务
- [ ] 项目脚手架搭建（React 18 + TypeScript + Vite）
- [ ] 配置 ESLint、Prettier、Husky
- [ ] 搭建基础路由结构（首页、编辑器、分享页）
- [ ] 引入 Zustand 状态管理
- [ ] 配置 TailwindCSS / Ant Design 组件库
- [ ] 搭建基础布局组件（Header、Sidebar、Main）

#### 后端任务
- [ ] 技术选型确认（Node.js + Express 或 Python + FastAPI）
- [ ] 项目结构初始化，配置 TypeScript/Python 环境
- [ ] 数据库设计（PostgreSQL）
  - 用户表（users）
  - 项目表（projects）
  - 任务表（tasks）
  - 分享链接表（share_links）
- [ ] Redis 环境配置（会话管理）
- [ ] RESTful API 基础框架搭建
- [ ] JWT 认证中间件实现

#### DevOps 任务
- [ ] Git 仓库创建与分支策略制定
- [ ] Docker 开发环境配置
- [ ] CI/CD 基础流水线搭建（GitHub Actions / GitLab CI）

---

### 第 3-4 周：手动填写模式 + 甘特图渲染引擎

#### 前端任务（P0 功能）
- [ ] 任务表单组件开发
  - 任务名称、阶段、负责方、开始/结束日期、说明
  - 新增/编辑/删除任务行
  - 日期选择器集成（Day.js）
- [ ] 阶段管理功能
  - 创建/重命名/删除阶段
  - 阶段折叠/展开交互
- [ ] 甘特图渲染引擎（核心）
  - Canvas/SVG 技术选型确认
  - 时间轴绘制（日/周/月粒度切换）
  - 任务条绘制与颜色编码
  - 今日标记线
  - 缩放与横向滚动
- [ ] 数据模型设计与状态同步

#### 后端任务
- [ ] 项目 CRUD API
  - POST /api/projects - 创建项目
  - GET /api/projects/:id - 获取项目详情
  - PUT /api/projects/:id - 更新项目
  - DELETE /api/projects/:id - 删除项目
- [ ] 任务 CRUD API
  - POST /api/projects/:id/tasks - 批量创建/更新任务
  - DELETE /api/tasks/:id - 删除任务
- [ ] 自动保存机制（30秒防抖）

---

### 第 5-6 周：分享功能 + 导出 + MVP 测试

#### 前端任务
- [ ] 只读分享页面开发
  - 无需登录即可查看
  - 精简版甘特图渲染
- [ ] PNG 导出功能
  - Canvas 转 2x 高清图片
  - 下载触发逻辑
- [ ] 分享链接生成与复制
- [ ] 响应式适配（桌面端优先）
- [ ] 访客模式（本地存储）

#### 后端任务
- [ ] 分享链接生成 API
  - POST /api/projects/:id/share - 生成唯一 token（≥24字符）
  - GET /api/share/:token - 获取分享项目数据
- [ ] 链接过期机制（默认永久，可设置过期时间）
- [ ] 访问统计（可选）

#### 测试与优化
- [ ] 10 名内测用户招募
- [ ] 端到端功能测试
- [ ] 性能优化（首屏加载 ≤ 2.5s）
- [ ] Bug 修复与体验优化

**M1 验收标准：**
✅ 甘特图正确渲染 100 个任务  
✅ 分享链接可正常访问  
✅ PNG 导出清晰完整

## 阶段二：M2 - AI 文件解析（第 7-10 周）

### 第 7-8 周：文件上传与 AI 集成

#### 前端任务
- [ ] 文件上传组件
  - 拖拽上传区域
  - 文件格式校验（.xlsx, .xls, .docx, .csv）
  - 文件大小限制（20MB）
  - 上传进度条
- [ ] 客户端文件预处理
  - 集成 SheetJS（Excel 解析）
  - 集成 Mammoth.js（Word 解析）
  - 多 Sheet 选择界面
- [ ] 加载状态与错误提示

#### 后端任务（核心）
- [ ] 文件上传接口
  - POST /api/upload - 接收文件并暂存
  - 文件类型与大小验证
- [ ] AI 解析服务集成
  - Claude API 接入（claude-sonnet-4-20250514）
  - 结构化 Prompt 设计
    ```
    解析以下项目文件，提取任务信息并输出 JSON：
    - 任务名称
    - 开始日期/结束日期
    - 负责方
    - 阶段分组
    - 任务说明
    ```
  - JSON Schema 定义与验证
- [ ] 异步任务队列（Bull/Celery）
  - 处理大文件解析
  - 进度状态推送（WebSocket/轮询）
- [ ] 文件清理机制（解析完成后删除）

---

### 第 9-10 周：字段确认界面 + 准确率优化

#### 前端任务
- [ ] 字段映射确认界面
  - 展示「原始列名 → 系统字段」映射
  - 下拉选择器修改映射
  - 实时预览解析结果
  - 低置信度字段高亮提示
- [ ] 历史文件记录（最近 10 次）
- [ ] 容错处理 UI
  - 明确的错误提示
  - 修复建议展示

#### 后端任务
- [ ] 字段映射调整 API
  - PUT /api/parse/:id/mapping - 更新字段映射
- [ ] AI Prompt 优化
  - 建立测试文件库（标准格式 + 边缘案例）
  - 迭代 Prompt 提升准确率至 90%+
- [ ] 多语言支持（中文优先）
- [ ] 格式模板库（引导用户上传规范文件）

#### 测试
- [ ] 准确率测试（≥90% 目标）
- [ ] 响应时间测试（10MB 文件 ≤ 15s）
- [ ] 边缘案例测试（复杂表头、多语言混排）

**M2 验收标准：**
✅ Excel/Word 文件识别准确率 ≥ 90%  
✅ 解析响应时间 ≤ 15 秒  
✅ 字段确认界面可正常调整映射

## 技术栈总结

### 前端
- React 18 + TypeScript
- 状态管理：Zustand
- 甘特图：Canvas/SVG 自研（参考 Frappe Gantt）
- 日期处理：Day.js
- 文件处理：SheetJS + Mammoth.js
- 实时通信：Socket.io-client

### 后端
- Node.js + Express（或 Python + FastAPI）
- 数据库：PostgreSQL + Redis
- AI：Claude API (claude-sonnet-4-20250514)
- 任务队列：Bull（Node.js）/ Celery（Python）
- 实时协作：Socket.io / WebSocket
- 文件存储：S3 兼容对象存储

### DevOps
- 容器化：Docker + Kubernetes
- CI/CD：GitHub Actions
- 监控：Sentry + Prometheus
- CDN：Cloudflare / AWS CloudFront

## 风险应对策略

### 1. AI 解析准确率不足
- 提供字段确认界面兜底
- 建立格式模板库
- 持续优化 Prompt
- 预留 2 周 buffer 用于准确率调优

### 2. 大文件解析超时
- 客户端预处理（SheetJS 提取纯数据）
- 后端异步任务 + 进度推送
- 文件大小限制 20MB

### 3. 甘特图性能瓶颈
- 虚拟化渲染（仅绘制可视区域）
- Canvas 分层渲染
- Web Worker 处理计算密集任务
- 预留第 17 周专门优化性能

### 4. 实时协作冲突
- 采用成熟的 OT/CRDT 算法
- 最后写入优胜 + 冲突通知
- 充分的并发测试（第 18 周）
