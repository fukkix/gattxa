## 🎉 新增功能

### 1. 邀请页面

完整的邀请接受页面，用户可以通过邀请链接查看邀请详情并接受邀请。

#### 功能特性

- ✅ 显示邀请详情
- ✅ 显示项目信息
- ✅ 显示邀请人信息
- ✅ 显示角色和权限说明
- ✅ 接受邀请
- ✅ 拒绝邀请
- ✅ 过期检测
- ✅ 状态检查
- ✅ 未登录用户引导

#### 页面内容

**邀请信息**:
- 项目名称
- 邀请人姓名和头像
- 邀请时间
- 角色名称和权限说明
- 邀请邮箱
- 过期时间

**状态提示**:
- 邀请已过期
- 邀请已被接受
- 邀请已失效
- 需要登录提示

**操作按钮**:
- 接受邀请（绿色主按钮）
- 拒绝邀请（灰色次按钮）
- 返回首页（底部链接）

---

### 2. 邀请流程优化

#### 未登录用户流程

1. 用户点击邀请链接
2. 打开邀请页面，查看邀请详情
3. 点击"接受邀请"按钮
4. 系统保存邀请 token 到 localStorage
5. 跳转到登录页面
6. 用户登录或注册
7. 登录成功后自动跳转回邀请页面
8. 自动接受邀请
9. 跳转到项目编辑器

#### 已登录用户流程

1. 用户点击邀请链接
2. 打开邀请页面，查看邀请详情
3. 点击"接受邀请"按钮
4. 系统调用接受邀请 API
5. 接受成功后跳转到项目编辑器

---

### 3. 邀请链接显示

邀请成功后显示邀请链接，方便复制分享。

#### 功能特性

- ✅ 邀请成功后弹窗显示链接
- ✅ 显示邀请邮箱
- ✅ 显示完整邀请 URL
- ✅ 可复制链接（通过弹窗）

## 🎯 API 文档

### 获取邀请信息（无需认证）

```http
GET /api/invitations/:token
```

**响应**:
```json
{
  "id": "invitation_id",
  "project_id": "project_id",
  "project_name": "项目名称",
  "email": "user@example.com",
  "role": "member",
  "invited_by_name": "张三",
  "invited_at": "2026-04-13T10:00:00Z",
  "expires_at": "2026-04-20T10:00:00Z",
  "status": "pending"
}
```

**错误响应**:
```json
{
  "error": "邀请不存在"
}
```

## 🚀 使用指南

### 发送邀请

1. 打开项目成员管理对话框
2. 点击"邀请成员"按钮
3. 输入邮箱地址
4. 选择角色
5. 点击"发送邀请"
6. 复制弹窗中显示的邀请链接
7. 将链接发送给对方（邮件、微信等）

### 接受邀请（未登录用户）

1. 收到邀请链接
2. 点击链接打开邀请页面
3. 查看邀请详情
4. 点击"接受邀请"按钮
5. 跳转到登录页面
6. 登录或注册账号
7. 自动跳转回邀请页面
8. 自动接受邀请
9. 进入项目编辑器

### 接受邀请（已登录用户）

1. 收到邀请链接
2. 点击链接打开邀请页面
3. 查看邀请详情
4. 点击"接受邀请"按钮
5. 直接进入项目编辑器

### 拒绝邀请

1. 打开邀请页面
2. 点击"拒绝"按钮
3. 确认拒绝
4. 返回首页

## 🎓 最佳实践

### 发送邀请

✅ **推荐做法**:
- 使用正确的邮箱地址
- 选择合适的角色
- 及时发送邀请链接
- 告知对方邀请有效期

❌ **不推荐做法**:
- 使用错误的邮箱
- 给错误的角色权限
- 不告知对方邀请信息
- 让邀请过期

### 接受邀请

✅ **推荐做法**:
- 及时接受邀请
- 使用正确的邮箱登录
- 查看角色权限说明
- 确认项目信息

❌ **不推荐做法**:
- 长时间不处理邀请
- 使用错误的邮箱登录
- 不查看权限说明
- 随意接受邀请
