## 🚀 快速启动步骤

### 1. 安装依赖（如果尚未完成）

**后端**:
```bash
cd gattxa/backend
npm install
```

**前端**:
```bash
cd gattxa/frontend  
npm install
```

### 2. 配置环境变量

**后端** (`gattxa/backend/.env`):
```env
PORT=3000
DATABASE_URL=postgresql://user:password@localhost:5432/ganttxa
JWT_SECRET=your-secret-key
CORS_ORIGIN=http://localhost:5173
```

**前端** (`gattxa/frontend/.env`):
```env
VITE_API_URL=http://localhost:3000
```

### 3. 启动服务

**启动数据库** (如果使用 Docker):
```bash
cd gattxa
docker-compose up -d postgres
```

**启动后端**:
```bash
cd gattxa/backend
npm run dev
```

**启动前端** (新终端):
```bash
cd gattxa/frontend
npm run dev
```

## 🧪 测试场景

### 场景 1: 基本连接测试

1. 打开浏览器访问 `http://localhost:5173`
2. 登录账号
3. 打开一个项目
4. **检查点**: 右上角应该显示绿色"实时"标签

**预期结果**: ✅ 显示"实时"标签表示 WebSocket 已连接

---

### 场景 2: 双窗口实时同步测试

1. 打开两个浏览器窗口（或使用隐身模式）
2. 两个窗口都登录（可以是同一账号或不同账号）
3. 两个窗口都打开同一个项目
4. **检查点**: 右下角应该显示 "2 人在线"

**测试步骤**:
- 在窗口 A 创建一个新任务
- 观察窗口 B 是否自动显示新任务

**预期结果**: 
- ✅ 窗口 B 在 1-2 秒内显示新任务
- ✅ 任务信息完全一致

---

### 场景 3: 任务编辑同步测试

1. 保持两个窗口打开同一项目
2. 在窗口 A 编辑一个任务（修改名称、日期等）
3. 保存修改
4. 观察窗口 B

**预期结果**: 
- ✅ 窗口 B 自动更新任务信息
- ✅ 甘特图同步刷新

---

### 场景 4: 任务删除同步测试

1. 在窗口 A 删除一个任务
2. 观察窗口 B

**预期结果**: 
- ✅ 窗口 B 自动移除该任务
- ✅ 甘特图同步更新

---

### 场景 5: 在线用户显示测试

1. 打开两个窗口，登录不同用户
2. 都打开同一项目
3. 点击右下角的在线用户组件

**预期结果**: 
- ✅ 显示两个用户的头像和姓名
- ✅ 每个用户右下角有绿色在线指示器

---

### 场景 6: 断线重连测试

1. 打开项目（显示"实时"标签）
2. 停止后端服务
3. 观察"实时"标签消失
4. 重启后端服务
5. 等待 1-5 秒

**预期结果**: 
- ✅ "实时"标签重新出现
- ✅ 自动重新加入项目房间
- ✅ 在线用户列表恢复

## ❌ 常见问题

### 问题 1: 不显示"实时"标签

**可能原因**:
- 后端未启动
- WebSocket 连接失败
- Token 无效

**解决方法**:
1. 检查后端是否运行
2. 查看浏览器控制台错误
3. 重新登录获取新 Token

---

### 问题 2: 任务不同步

**可能原因**:
- WebSocket 连接断开
- 事件监听器未设置

**解决方法**:
1. 检查"实时"标签是否显示
2. 刷新页面重新连接
3. 查看浏览器控制台日志

---

### 问题 3: 在线用户不显示

**可能原因**:
- 未成功加入项目房间
- 权限不足

**解决方法**:
1. 检查用户是否有项目访问权限
2. 查看后端日志
3. 刷新页面
