# M4 实时协作功能测试指南

## 📋 测试准备

### 环境要求
- Node.js 18+
- PostgreSQL 数据库
- 2 个或更多浏览器窗口/标签页

### 启动服务

**后端**:
```bash
cd backend
npm install
npm run dev
```

**前端**:
```bash
cd frontend
npm install
npm run dev
```

## 🧪 测试场景

### 场景 1：WebSocket 连接测试

**目的**: 验证 WebSocket 连接能否正常建立

**步骤**:
1. 打开浏览器，访问 `http://localhost:5173`
2. 登录账号
3. 打开或创建一个项目
4. 观察编辑器页面右上角是否显示"实时"标签（绿色脉冲）

**预期结果**:
- ✅ 显示"实时"标签
- ✅ 浏览器控制台显示 "✅ WebSocket 已连接"
- ✅ 后端控制台显示用户连接信息

**失败排查**:
- 检查后端是否正常运行
- 检查 JWT Token 是否有效
- 检查浏览器控制台错误信息

---

### 场景 2：在线用户显示测试

**目的**: 验证在线用户列表能否正确显示

**步骤**:
1. 窗口 A：登录用户 A，打开项目
2. 窗口 B：登录用户 B，打开同一项目
3. 观察右下角在线用户组件

**预期结果**:
- ✅ 窗口 A 显示 "2 人在线"
- ✅ 窗口 B 显示 "2 人在线"
- ✅ 点击展开，显示两个用户的头像和姓名
- ✅ 用户头像右下角有绿色在线指示器

**失败排查**:
- 检查 WebSocket 连接是否成功
- 检查浏览器控制台是否有错误
- 检查用户是否成功加入项目房间

---

### 场景 3：实时任务创建测试

**目的**: 验证任务创建能否实时同步

**步骤**:
1. 窗口 A 和 B 同时打开同一项目
2. 在窗口 A 中点击"新建任务"
3. 填写任务信息并保存
4. 观察窗口 B 的任务列表

**预期结果**:
- ✅ 窗口 A 立即显示新任务
- ✅ 窗口 B 在 1-2 秒内显示新任务
- ✅ 两个窗口的任务信息一致
- ✅ 甘特图同步更新

**失败排查**:
- 检查浏览器控制台是否显示 "收到任务创建事件"
- 检查 WebSocket 连接状态
- 检查任务数据是否正确

---

### 场景 4：实时任务更新测试

**目的**: 验证任务更新能否实时同步

**步骤**:
1. 窗口 A 和 B 同时打开同一项目
2. 在窗口 A 中编辑一个任务
3. 修改任务名称、日期或其他信息
4. 保存修改
5. 观察窗口 B 的任务列表

**预期结果**:
- ✅ 窗口 A 立即显示更新
- ✅ 窗口 B 在 1-2 秒内显示更新
- ✅ 两个窗口的任务信息一致
- ✅ 甘特图同步更新

**失败排查**:
- 检查浏览器控制台是否显示 "收到任务更新事件"
- 检查任务 ID 是否正确
- 检查更新数据是否完整

---

### 场景 5：实时任务删除测试

**目的**: 验证任务删除能否实时同步

**步骤**:
1. 窗口 A 和 B 同时打开同一项目
2. 在窗口 A 中删除一个任务
3. 确认删除
4. 观察窗口 B 的任务列表

**预期结果**:
- ✅ 窗口 A 立即移除任务
- ✅ 窗口 B 在 1-2 秒内移除任务
- ✅ 甘特图同步更新

**失败排查**:
- 检查浏览器控制台是否显示 "收到任务删除事件"
- 检查任务 ID 是否正确

---

### 场景 6：实时评论测试

**目的**: 验证评论能否实时同步

**步骤**:
1. 窗口 A 和 B 同时打开同一项目
2. 在窗口 A 中打开一个任务的评论
3. 在窗口 B 中也打开同一任务的评论
4. 在窗口 A 中添加评论
5. 观察窗口 B 的评论列表

**预期结果**:
- ✅ 窗口 A 立即显示新评论
- ✅ 窗口 B 在 1-2 秒内显示新评论（需要刷新评论列表）
- ✅ 评论内容一致

**注意**: 当前版本评论需要手动刷新，后续版本将实现自动刷新

---

### 场景 7：用户加入/离开测试

**目的**: 验证用户加入和离开能否正确通知

**步骤**:
1. 窗口 A：用户 A 打开项目
2. 窗口 B：用户 B 打开同一项目
3. 观察窗口 A 的在线用户列表
4. 关闭窗口 B
5. 观察窗口 A 的在线用户列表

**预期结果**:
- ✅ 用户 B 加入时，窗口 A 显示 "2 人在线"
- ✅ 用户 B 离开时，窗口 A 显示 "1 人在线"
- ✅ 在线用户列表实时更新

---

### 场景 8：断线重连测试

**目的**: 验证断线后能否自动重连

**步骤**:
1. 打开项目
2. 观察"实时"标签显示
3. 停止后端服务（模拟断线）
4. 观察"实时"标签消失
5. 重启后端服务
6. 等待 1-5 秒
7. 观察"实时"标签重新出现

**预期结果**:
- ✅ 断线时"实时"标签消失
- ✅ 重连后"实时"标签重新出现
- ✅ 自动重新加入项目房间
- ✅ 在线用户列表恢复正常

**失败排查**:
- 检查浏览器控制台重连日志
- 检查重连次数是否超过限制（5次）
- 检查后端是否正常启动

---

### 场景 9：心跳机制测试

**目的**: 验证心跳机制能否保持连接

**步骤**:
1. 打开项目
2. 保持页面打开 5 分钟
3. 观察连接状态
4. 尝试创建任务

**预期结果**:
- ✅ 连接保持活跃
- ✅ "实时"标签持续显示
- ✅ 任务创建正常同步

**失败排查**:
- 检查浏览器控制台是否有心跳日志
- 检查后端是否收到心跳消息

---

### 场景 10：多用户并发编辑测试

**目的**: 验证多用户同时编辑的表现

**步骤**:
1. 打开 3 个浏览器窗口（用户 A、B、C）
2. 所有窗口打开同一项目
3. 用户 A 创建任务 1
4. 用户 B 创建任务 2
5. 用户 C 更新任务 1
6. 观察所有窗口的任务列表

**预期结果**:
- ✅ 所有窗口显示 "3 人在线"
- ✅ 所有窗口都能看到任务 1 和任务 2
- ✅ 所有窗口都能看到任务 1 的更新
- ✅ 操作顺序正确

**注意**: 当前版本尚未实现冲突检测，如果同时编辑同一任务可能出现覆盖

## 🐛 常见问题

### 问题 1：WebSocket 连接失败

**症状**: 不显示"实时"标签

**可能原因**:
- 后端未启动
- JWT Token 无效
- CORS 配置错误

**解决方法**:
1. 检查后端是否运行
2. 重新登录获取新 Token
3. 检查 CORS 配置

---

### 问题 2：在线用户不显示

**症状**: 在线用户列表为空

**可能原因**:
- 未成功加入项目房间
- 权限不足

**解决方法**:
1. 检查浏览器控制台错误
2. 检查用户是否有项目访问权限
3. 刷新页面重新加入

---

### 问题 3：任务不同步

**症状**: 一个窗口的操作在另一个窗口不显示

**可能原因**:
- WebSocket 连接断开
- 事件监听器未正确设置
- 项目 ID 不匹配

**解决方法**:
1. 检查"实时"标签是否显示
2. 刷新页面重新连接
3. 检查浏览器控制台日志

---

### 问题 4：重连失败

**症状**: 断线后无法重连

**可能原因**:
- 超过最大重连次数（5次）
- Token 过期

**解决方法**:
1. 刷新页面
2. 重新登录
3. 检查后端是否正常

## 🔍 调试技巧

### 查看 WebSocket 连接状态

**浏览器控制台**:
```javascript
// 检查连接状态
websocketService.isConnected()

// 获取当前项目 ID
websocketService.getCurrentProjectId()
```

### 查看在线用户

**后端控制台**:
```javascript
// 在 websocket.ts 中添加
console.log('在线用户:', Array.from(onlineUsers.entries()))
```

### 查看事件日志

**浏览器控制台**:
- 所有 WebSocket 事件都会打印日志
- 查找 "收到任务创建事件"、"收到任务更新事件" 等

**后端控制台**:
- 查找 "用户连接"、"用户加入项目" 等日志
