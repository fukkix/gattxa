## 📝 需要执行的操作

### 1. 运行数据库迁移 ⏳

```bash
cd gattxa
psql $DATABASE_URL -f database/migrations/004_add_version_to_tasks.sql
```

或者手动执行 SQL：

```sql
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS version INTEGER DEFAULT 1 NOT NULL;
UPDATE tasks SET version = 1 WHERE version IS NULL;
CREATE INDEX IF NOT EXISTS idx_tasks_version ON tasks(id, version);
```

### 2. 重启后端服务器 ⏳

```bash
cd gattxa/backend
# 在运行服务器的终端按 Ctrl+C
npm run dev
```

### 3. 重启前端服务器 ⏳

```bash
cd gattxa/frontend
# 在运行服务器的终端按 Ctrl+C
npm run dev
```

## 🧪 测试准备

### 测试场景

#### 场景 1: 触发冲突
1. 打开两个浏览器窗口
2. 都打开同一个项目
3. 窗口 A 编辑一个任务并保存
4. 窗口 B 编辑同一个任务并保存
5. 窗口 B 应该看到冲突对话框

#### 场景 2: 保留本地更改
1. 在冲突对话框选择"保留我的更改"
2. 点击"解决冲突"
3. 检查任务是否使用本地值

#### 场景 3: 使用远程更改
1. 在冲突对话框选择"使用远程更改"
2. 点击"解决冲突"
3. 检查任务是否使用远程值

#### 场景 4: 手动合并
1. 在冲突对话框选择"手动合并"
2. 逐字段选择要保留的值
3. 点击"解决冲突"
4. 检查任务是否正确合并
