# GanttXa 本地开发指南

**日期**: 2026-04-13  
**适用**: 本地开发环境

---

## 📋 前置要求

### 必需软件

- **Node.js**: v18.20.8 或更高版本
- **npm**: v9+ 或 yarn
- **PostgreSQL**: v15+ (或使用 Docker)
- **Git**: 最新版本

### 可选软件

- **Docker Desktop**: 用于运行数据库（推荐）
- **nvm**: Node.js 版本管理工具

## 🚀 快速启动（5 分钟）

### 方法 1: 使用 Docker（推荐）

**优势**: 无需手动安装 PostgreSQL，一键启动所有服务

```bash
# 1. 克隆项目
git clone https://github.com/fukkix/ganttxa.git
cd ganttxa

# 2. 启动数据库
docker-compose up -d postgres

# 3. 启动后端
cd backend
npm install
npm run dev

# 4. 启动前端（新终端）
cd frontend
npm install
npm run dev
```

**访问地址**:
- 前端: http://localhost:5173
- 后端: http://localhost:3000

---

### 方法 2: 手动安装所有服务

**适用**: 不使用 Docker

#### 步骤 1: 安装 Node.js 18

```bash
# 使用 nvm（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc  # 或 ~/.zshrc

# 安装 Node.js 18
nvm install 18
nvm use 18

# 验证版本
node -v  # 应该显示 v18.x.x
```

#### 步骤 2: 安装 PostgreSQL

**macOS**:
```bash
brew install postgresql@15
brew services start postgresql@15
```

**Ubuntu/Debian**:
```bash
sudo apt update
sudo apt install postgresql-15
sudo systemctl start postgresql
```

**Windows**:
下载并安装: https://www.postgresql.org/download/windows/

#### 步骤 3: 创建数据库

```bash
# 连接到 PostgreSQL
psql postgres

# 创建数据库和用户
CREATE DATABASE ganttxa;
CREATE USER ganttxa_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE ganttxa TO ganttxa_user;
\q
```

#### 步骤 4: 配置环境变量

**后端** (`backend/.env`):
```bash
PORT=3000
DATABASE_URL=postgresql://ganttxa_user:your_password@localhost:5432/ganttxa
JWT_SECRET=your_jwt_secret_at_least_32_characters_long
CORS_ORIGIN=http://localhost:5173
```

**前端** (`frontend/.env`):
```bash
VITE_API_URL=http://localhost:3000
```

#### 步骤 5: 初始化数据库

```bash
# 运行初始化脚本
cd ganttxa
psql -U ganttxa_user -d ganttxa -f database/init.sql

# 运行迁移
psql -U ganttxa_user -d ganttxa -f database/migrations/001_add_comments_table.sql
psql -U ganttxa_user -d ganttxa -f database/migrations/002_add_comment_reads_table.sql
psql -U ganttxa_user -d ganttxa -f database/migrations/003_add_project_members_table.sql
psql -U ganttxa_user -d ganttxa -f database/migrations/004_add_version_to_tasks.sql
```

#### 步骤 6: 安装依赖并启动

```bash
# 后端
cd backend
npm install
npm run dev

# 前端（新终端）
cd frontend
npm install
npm run dev
```

## 📝 详细步骤说明

### 1. 克隆项目

```bash
git clone https://github.com/fukkix/ganttxa.git
cd ganttxa
```

### 2. 配置环境变量

#### 后端环境变量

创建 `backend/.env` 文件：

```bash
cd backend
cp .env.example .env
nano .env
```

**必需配置**:
```env
# 服务器端口
PORT=3000

# 数据库连接
DATABASE_URL=postgresql://ganttxa_user:your_password@localhost:5432/ganttxa

# JWT 密钥（至少 32 个字符）
JWT_SECRET=your_super_secret_jwt_key_at_least_32_characters_long

# CORS 来源（前端地址）
CORS_ORIGIN=http://localhost:5173
```

#### 前端环境变量

创建 `frontend/.env` 文件：

```bash
cd frontend
cp .env.example .env
nano .env
```

**必需配置**:
```env
# 后端 API 地址
VITE_API_URL=http://localhost:3000
```

### 3. 启动数据库

#### 使用 Docker（推荐）

```bash
# 在项目根目录
docker-compose up -d postgres

# 查看数据库日志
docker-compose logs -f postgres

# 停止数据库
docker-compose down
```

#### 使用本地 PostgreSQL

```bash
# macOS
brew services start postgresql@15

# Ubuntu/Debian
sudo systemctl start postgresql

# 验证运行状态
psql --version
```

### 4. 初始化数据库

```bash
# 方法 1: 使用 Docker（如果数据库在 Docker 中）
docker-compose exec postgres psql -U ganttxa_user -d ganttxa -f /docker-entrypoint-initdb.d/init.sql

# 方法 2: 使用本地 psql
cd ganttxa
psql -U ganttxa_user -d ganttxa -f database/init.sql

# 运行所有迁移
for file in database/migrations/*.sql; do
  psql -U ganttxa_user -d ganttxa -f "$file"
done
```

### 5. 安装后端依赖

```bash
cd backend
npm install

# 如果遇到权限问题
sudo npm install

# 验证安装
npm list
```

**预期输出**: 应该看到 355 个包安装成功

### 6. 启动后端服务

```bash
cd backend
npm run dev
```

**成功标志**:
```
✅ WebSocket 服务已初始化
🚀 Server running on http://localhost:3000
📝 Environment: development
📁 Upload directory: ./uploads
🔌 WebSocket ready
```

### 7. 安装前端依赖

**打开新终端**:

```bash
cd frontend
npm install

# 验证安装
npm list
```

### 8. 启动前端服务

```bash
cd frontend
npm run dev
```

**成功标志**:
```
VITE v5.4.21  ready in XXX ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
➜  press h + enter to show help
```

## 🔍 故障排查

### 问题 1: Node.js 版本不对

**错误**: `Error: The engine "node" is incompatible`

**解决**:
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装 Node.js 18
nvm install 18
nvm use 18

# 验证
node -v
```

### 问题 2: 数据库连接失败

**错误**: `Error: connect ECONNREFUSED`

**解决**:
```bash
# 检查数据库是否运行
docker-compose ps
# 或
pg_isready -h localhost -p 5432

# 检查连接字符串
echo $DATABASE_URL

# 重启数据库
docker-compose restart postgres
```

### 问题 3: 端口被占用

**错误**: `Error: listen EADDRINUSE: address already in use :::3000`

**解决**:
```bash
# 查找占用端口的进程
lsof -i :3000
# 或
netstat -tulpn | grep :3000

# 杀死进程
kill -9 <PID>

# 或使用不同端口
PORT=3001 npm run dev
```

### 问题 4: npm install 失败

**错误**: `npm ERR! code EACCES`

**解决**:
```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules
rm -rf node_modules package-lock.json

# 重新安装
npm install

# 如果还是失败，使用 sudo（不推荐）
sudo npm install
```

### 问题 5: 前端无法连接后端

**错误**: `Network Error` 或 `CORS Error`

**解决**:
```bash
# 1. 检查后端是否运行
curl http://localhost:3000

# 2. 检查 CORS 配置
# backend/.env
CORS_ORIGIN=http://localhost:5173

# 3. 检查前端 API 地址
# frontend/.env
VITE_API_URL=http://localhost:3000

# 4. 重启两个服务
```

### 问题 6: WebSocket 连接失败

**错误**: `WebSocket connection failed`

**解决**:
```bash
# 1. 检查后端日志
# 应该看到 "✅ WebSocket 服务已初始化"

# 2. 检查浏览器控制台
# 按 F12 查看错误信息

# 3. 检查 Token
# 确保已登录，localStorage 中有 token

# 4. 重启后端服务
cd backend
npm run dev
```

## 🎯 开发工作流

### 1. 每天开始工作

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 启动数据库
docker-compose up -d postgres

# 3. 启动后端（终端 1）
cd backend
npm run dev

# 4. 启动前端（终端 2）
cd frontend
npm run dev

# 5. 打开浏览器
open http://localhost:5173
```

### 2. 开发新功能

```bash
# 1. 创建新分支
git checkout -b feature/your-feature-name

# 2. 开发代码
# ...

# 3. 测试功能
# ...

# 4. 提交代码
git add .
git commit -m "feat: 添加新功能"

# 5. 推送到远程
git push origin feature/your-feature-name
```

### 3. 结束工作

```bash
# 1. 提交所有更改
git add .
git commit -m "chore: 保存工作进度"
git push

# 2. 停止服务
# 按 Ctrl+C 停止前端和后端

# 3. 停止数据库（可选）
docker-compose down
```
