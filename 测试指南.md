## 🧪 快速测试步骤

### 1️⃣ 基本连接测试（1分钟）

1. 打开浏览器访问: http://localhost:5173
2. 登录你的账号
3. 打开任意一个项目
4. **检查**: 右上角是否显示绿色的 **"实时"** 标签

✅ **成功标志**: 看到"实时"标签，说明 WebSocket 已连接

---

### 2️⃣ 双窗口实时同步测试（3分钟）

1. **打开第二个窗口**:
   - 方法 1: 新建隐身窗口（Ctrl+Shift+N 或 Cmd+Shift+N）
   - 方法 2: 使用不同浏览器

2. **两个窗口都登录并打开同一个项目**

3. **检查在线用户**:
   - 右下角应该显示 **"2 人在线"**
   - 点击展开，可以看到两个用户的头像

4. **测试任务同步**:
   - 在窗口 A 创建一个新任务
   - 观察窗口 B 是否在 1-2 秒内自动显示这个任务

✅ **成功标志**: 
- 显示 "2 人在线"
- 窗口 B 自动显示新任务

---

### 3️⃣ 任务编辑同步测试（2分钟）

1. 保持两个窗口打开
2. 在窗口 A 编辑一个任务（修改名称或日期）
3. 保存修改
4. 观察窗口 B 是否自动更新

✅ **成功标志**: 窗口 B 自动更新任务信息

---

### 4️⃣ 任务删除同步测试（1分钟）

1. 在窗口 A 删除一个任务
2. 观察窗口 B

✅ **成功标志**: 窗口 B 自动移除该任务

---

### 5️⃣ 断线重连测试（2分钟）

1. 打开项目（显示"实时"标签）
2. 在终端按 Ctrl+C 停止后端服务
3. 观察"实时"标签消失
4. 重新启动后端服务:
   ```bash
   cd gattxa/backend
   npm run dev
   ```
5. 等待 1-5 秒

✅ **成功标志**: "实时"标签重新出现

## ❌ 常见问题

### 问题 1: 不显示"实时"标签

**解决方法**:
1. 检查后端服务是否运行
2. 刷新页面
3. 查看浏览器控制台是否有错误

### 问题 2: 任务不同步

**解决方法**:
1. 检查"实时"标签是否显示
2. 刷新两个窗口
3. 查看浏览器控制台日志

### 问题 3: 在线用户不显示

**解决方法**:
1. 确保两个窗口都打开了同一个项目
2. 刷新页面
3. 检查用户是否有项目访问权限
